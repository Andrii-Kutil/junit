В параметризованные тесты можно передавать и другие коллекции, к примеру, csv, енамы, результат метода

Параметризованные тесты принимают параметр из вне.

Можно создавать вложенные тесты, использовать аннотацию для встроенных классов тестов.

С помощью плагина можно сгенерировать авто тесты.

@Spy используется для подмены объекта и возможность использовать весь функционал на подменном

Когда указываешь when then... то метод возвращает сразу то что ты указал, будто этот метод вызывался.
doReturn when отличается тем что это выражение описывает условие и когда в дальнейшем этот метод вызывается то только тогда вызовется метод.

verify() метод для проверки какой метод вызывался, сколько раз и так далее

@Spy vs @Mock - мок это лиш скелет, к примеру в него нельзя ничего положить, а спу эт полноценная подмена обьекта

@InjectMocks позволяет сделать Dependency Injection с помощью Mock полей

answer удобен потому что можно получить информацию к примеру какие аргументы были переданные в метод и
если это void то можно получить дополнительную информацию
